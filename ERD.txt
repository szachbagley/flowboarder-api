LEGEND
──────
*  Primary Key (PK)
#  Foreign Key (FK)
?  Optional/Nullable attribute
1  One side of relationship
∞  Many side of relationship
>  Relationship direction


RELATIONSHIPS
─────────────
USER ──────────────────< PROJECT           (One User has many Projects)
PROJECT ───────────────< CHARACTER         (One Project has many Characters)
PROJECT ───────────────< SETTING           (One Project has many Settings)
PROJECT ───────────────< PANEL             (One Project has many Panels)
CHARACTER ─────────────< CHARACTER_VARIANT (One Character has many Variants)
SETTING ───────────────< PANEL             (One Setting is used in many Panels)
CHARACTER_VARIANT >────< PANEL             (Many-to-Many via PANEL_CHARACTER)


ATTRIBUTE NOTES
───────────────
reference_img (Optional)
  - CHARACTER: Reference image for the base character design
  - CHARACTER_VARIANT: Reference image for this specific variant
  - SETTING: Reference image for the setting/environment
  - PANEL: Reference image for composition/style guidance

image_link (PANEL only)
  - The AI-generated storyboard image for this panel


JUNCTION TABLE NOTES
────────────────────
PANEL_CHARACTER resolves the many-to-many relationship between
panels and character variants. Each record represents a specific
character variant appearing in a specific panel.

Composite Primary Key: (panel_id, character_variant_id)
